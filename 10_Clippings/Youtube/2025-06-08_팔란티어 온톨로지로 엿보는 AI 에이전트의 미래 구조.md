# 개요

본 영상은 팔란티어(Palantir)가 제안하는 온톨로지 기반의 AI 시스템 구조와 그것이 조직 내 워크플로우 자동화에 어떻게 기여하는지를 설명합니다. 단일 작업 단위에서 시작해 전체 업무 흐름을 이해하는 '에이전트 시스템'으로 확장되기 위해 온톨로지 개념이 필수적임을 강조하며, 실제 구축 사례와 도입 전략을 공유합니다.

# 내용

### 팔란티어와 온톨로지 개념
- [(00:00)](https://www.youtube.com/watch?v=ctKz2bkgkPQ&t=0s)
- 팔란티어는 조직의 워크플로우를 노드(업무 단위)와 엣지(관계)로 모델링하여 AI와 연결하는 온톨로지 기반 시스템을 구축
- 온톨로지는 조직의 디지털 트윈으로, 현실의 업무 흐름을 그래프 구조로 표현함

### 온톨로지 기반 분석의 강점
- [(01:31)](https://www.youtube.com/watch?v=ctKz2bkgkPQ&t=91s)
- 특정 노드에서 문제가 발생했을 때, 연결된 이전-이후 노드를 통해 근본 원인을 파악 가능
- 예: 배송 문제 발생 → 구매단계 등 상위 노드 분석으로 원인 추적

### 온톨로지 구축의 필요성과 전략
- [(03:10)](https://www.youtube.com/watch?v=ctKz2bkgkPQ&t=190s)
- 데이터 정제와 연결성 확보가 선행되어야 전체 흐름을 이해하는 에이전트 개발 가능
- 노드별 데이터만 처리할 경우, 전체 문맥(Context)을 AI가 이해하지 못함

### 에이전트 설계와 온톨로지 활용법
- [(04:34)](https://www.youtube.com/watch?v=ctKz2bkgkPQ&t=274s)
- 프롬프트 설계 시 온톨로지 기반의 외부 이벤트를 매개변수로 제공해야 정확한 응답 가능
- 데이터 간 인과관계를 사전에 정의해두어야 AI가 유기적인 판단 가능

### 온톨로지에 대한 현실적인 접근
- [(06:14)](https://www.youtube.com/watch?v=ctKz2bkgkPQ&t=374s)
- 현재는 문서 파싱에 집중하더라도, 미래 확장을 고려한 구조 설계가 필요
- 향후 요구되는 에이전트 시스템은 온톨로지 기반 구조가 유리함

### 팔란티어의 SI 방식과 FD 제도
- [(07:49)](https://www.youtube.com/watch?v=ctKz2bkgkPQ&t=469s)
- Palantir는 Forward Deployed Engineer(FD)를 파견해 현업 경험을 바탕으로 맞춤형 온톨로지 설계
- 최근에는 LLM을 통해 이 과정을 자동화하려는 시도도 있음
- 일반 SI 업체와 달리, 업무 이해 기반의 솔루션 설계에 초점을 둠

### 도입 유망 산업과 국내 사례
- [(09:15)](https://www.youtube.com/watch?v=ctKz2bkgkPQ&t=555s)
- 물류, 공급망, 제조 등 복잡한 상호관계가 명확한 산업군에서 특히 유효
- 삼성전자(반도체), 삼양(공급망) 등의 국내 도입 사례 존재
- 재고관리와 운송 파이프라인 등 고정된 엔티티에서 디지털 트윈 적용 용이

### 실무적 인사이트와 에이전트 기획
- [(13:19)](https://www.youtube.com/watch?v=ctKz2bkgkPQ&t=799s)
- 에이전트 시스템 설계 시, 현업 흐름을 이해하고 설계할 수 있는 인재 또는 조직 내 담당자 필요
- 단순 기능 구현이 아닌 ‘문제 해결형 에이전트’ 설계가 중요
- 기술 도입 이전에 업무 이해 기반의 아키텍처 설계가 우선되어야 함

# 결론
팔란티어의 온톨로지 시스템은 전체 조직의 흐름을 이해하고 분석하는 에이전트 설계의 기반이 된다. 특히 제조, 물류, 공급망 같은 복잡한 관계 기반 산업에서 효과적이며, LLM과 결합으로 접근성과 자동화 수준이 높아질 것이다. 지금은 단일 작업 중심 자동화에 집중하더라도, 향후 온톨로지 기반 설계를 염두에 두는 것이 미래 경쟁력 확보에 중요하다.
