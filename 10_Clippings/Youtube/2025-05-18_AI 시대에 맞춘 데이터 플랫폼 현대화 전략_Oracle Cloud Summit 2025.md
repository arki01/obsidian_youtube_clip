# 개요

LLM, 벡터 검색, 자연어 질의, 로우코드 앱 등 최신 AI 기술을 데이터 중심으로 실무에 어떻게 접목할 것인지 설명

# 내용

## 1. 데이터 플랫폼의 핵심은 "내 데이터"
(00:00 ~ 01:53)

- 아무리 강력한 LLM도 양질의 내부 데이터가 없으면 의미 없음.
- 기업이 AI로부터 가치를 얻으려면 데이터 취득 → 저장 → 분석 → 활용 전 주기를 정비해야 함.
- 예시: GPT도 학습할 데이터가 부족해지고 있다는 말이 나오는 상황에서, 기업 내부 데이터는 전략적 자산.

## 2. Oracle AI Stack 및 멀티클라우드 전략
(01:53 ~ 05:04)

- Oracle은 Fusion Apps (SaaS), Middleware, IaaS, Data 플랫폼을 통합한 AI 스택 제공.
- 멀티클라우드 환경에서의 유연한 배포 지원 (온프레미스, AWS, Azure, GCP 등).
- 포인트: 데이터가 어디 있든 연결하고 분석하는 능력이 핵심임.

---

## 3. Converged Database - 모든 타입을 하나로
(08:38 ~ 10:24)

- 관계형(RDB), JSON, Graph, Spatial, Vector 데이터까지 단일 DB에서 처리 가능.
- 장점:
  - 보안 및 정합성 확보
  - 시스템 운영 효율성 증대
- 예시: IoT 센서 데이터 + 지리정보 + 고객데이터 통합 분석

---

## 4. 벡터 검색 (Semantic Search)
(13:10 ~ 16:56)

- 벡터란? 언어/이미지의 의미를 수치로 표현한 다차원 값
- 사용 사례:
  - 유저가 찍은 사진(지붕 모양 등) → 유사 매물 추천
  - 문장/문서 간 의미 기반 유사성 분석
- Oracle은 기존 DB 위에서 벡터 검색을 실시간 OLTP 환경에서도 지원

---

## 5. Select AI - 자연어 → SQL 변환
(18:42 ~ 21:39)

- 일반 사용자가 자연어로 질의 → SQL로 자동 변환 → 결과 반환
- 한글 지원 가능, LLM 연동됨
- 비개발자도 데이터 질의 가능
- 예: “2020년 이후 인기 있는 영화 알려줘” → SQL 자동 생성

## 6. Apex + AI로 로우코드 앱 생성
(22:42 ~ 24:44)

- 자연어로 “병원 진료 관리 앱 만들어줘” → 앱 자동 생성
- DB 테이블 생성, UI 구성, 기능 설정까지 자동화됨
- 생성된 앱에 AI 챗봇 연동도 가능
- 비개발자도 업무용 앱을 손쉽게 만들 수 있음

## 7. OAC - AI 리포트 자동 생성 (Oracle Analytics Cloud)
(25:41 ~ 27:30)

- 자연어로 리포트 생성 → 그래프 및 대시보드 자동 구성
- 현업 사용자도 데이터 기반 의사결정 가능
- 분석가가 아닌 사람도 셀프 BI 환경을 이용할 수 있음

---

## 8. AI 기반 고객지원 시나리오 예시
(28:35 ~ 30:33)

- 예: 고객의 발전기 고장 문의 → DB에서 고객 이력 + 제품 정보 + 고장 사례 검색
- 고장 부위 예측 → 부품 재고 위치 탐색 → 고객에게 안내
- 사용 기술: RDB + Knowledge Base + Inventory + LLM + Vector Search
