# 개요

Edge 파트는 AI 기술을 현장의 실질적 성과로 전환하는 실행 조직(AI-on-Site Hub) 으로서,
AI Board 내의 기술·데이터·전략 파트와 현장 조직을 연결해 AI 내재화(Internalization) 를 주도합니다.

그동안 S.CON을 중심으로 CQSS 전 영역의 디지털 관리체계를 운영하며
현장 데이터를 표준화하고, 데이터 기반 의사결정의 토대를 다져왔습니다.
이제 Edge는 이러한 경험을 바탕으로, AI가 현장에서 실제로 작동하는 단계로 발전시키고자 합니다.

운영 전략의 핵심은 **“직접 수행이 아닌 조율과 확산”**에 있습니다.
Edge는 기술을 개발하기보다,

1. 현장의 Pain Point를 정의하고,

2. 가장 적합한 AI 솔루션을 연계·검증하며,

3. 그 결과를 전사로 확산하는 조율자(Orchestrator) 의 역할을 수행합니다.

이를 위해 AI Board 내부 협업(Alpha·Build·Foundation)뿐만 아니라,
DX조직·품질본부·안전본부 등 타 부서 및 주요 현장과의 공동 추진체계(Co-Lead 모델) 를 구축하여
소규모 핵심팀이지만 대규모 영향력(Leverage) 을 확보하는 네트워크형 운영을 지향합니다.

결과적으로 Edge 파트는
데이터 수집 → AI 실증 → 자동화 → 통합관리 로 이어지는 전 과정을 주도하며,
AI Board의 비전인 “AI가 작동하는 현장, AI-on-Site” 를 실현하는 중심 역할을 수행할 것입니다.

# 내용

## 추진 핵심
1. Edge는 기획과 조율 중심으로 움직인다.
	- Edge 파트의 전략적 역할은 “직접 수행”보다 “AI 현장 혁신의 허브(허락된 조율자)”
	- 기술을 직접 개발하는 팀이 아니라, 기술과 현장을 연결하고 조율하는 실행 허브(Execution Hub) 역할을 수행
	- 기술 상세는 Build, 인프라는 Foundation에 위임
2. 각 PoC에 ‘공동 리더십 구조(Co-Lead)’를 만든다.
	- 현장 담당자와 Edge 담당자가 1:1 매칭
3. 협력 조직에 ‘Win 포인트’를 제공한다.
	- 단순 지원요청이 아니라, “성과 공유 구조”로 설계
4. 협업 프로세스를 단순화한다.
	- 의사결정 라인을 최소화하고, Board 내 승인체계 활용

## 파트 내 역할 분담
- 현실적으로 4명이라면, 각자의 전문역량을 축으로 한 역할 분화 + 공동 PoC 체계로 운영하는 게 가장 효율적

| 역할군                      | 담당자 방향  | 주요 역할                                                            | 협업 포인트                             |
| ------------------------ | ------- | ---------------------------------------------------------------- | ---------------------------------- |
| **AI Solution 리드**           | 리더 / 총괄 | - Edge 전체 방향 설정<br>- AI Board 및 타 파트 협의<br>- 현장 PoC 기획 및 ROI 관리  | Alpha와 PoC 선정 / Foundation과 KPI 협의 |
| **Data & Infra 담당**          | 엔지니어형   | - 센서·IoT·영상 데이터 수집체계 설계<br>- 데이터 품질 검증 및 표준화<br>- Data Lake 구축   | Foundation 파트와 연계 (E2E Dataflow)   |
| **AI Application 담당**        | 기획/연구형  | - Vision·Agent·예측모델 PoC 기획 및 모델링<br>- AI 모델 성능 분석, 적용 전략 수립      | Build 파트와 협업 (모델 내재화)              |
| **Robotics & Automation 담당** | 현장/기술형  | - 로봇·드론·자동화 실증<br>- Safety/Quality 자동화 솔루션 운영<br>- 현장 데이터 피드백 확보 | Alpha 파트와 확산사례 공유                  |

## 수행전략 3단계 로드맵 (내부 협업 프레임)

| 단계                                | 전략 초점                                                   | 주요 액션                                                         | 협업 파트                                      |
| --------------------------------- | ------------------------------------------------------- | ------------------------------------------------------------- | ------------------------------------------ |
| **1단계 (Q1-Q2)**<br>“현장 데이터 기반 구축” | - IoT·카메라·로봇 등 데이터 수집체계 확보<br>- Pain Point별 AI 적용 기회 도출 | - 현장별 Data Lake 파일럿 구축<br>- AI Board 내 Pain Point 매핑 워크숍      | Foundation (데이터 표준), Alpha (Pain Point 선정) |
| **2단계 (Q2-Q3)**<br>“PoC 실행 및 검증”  | - Vision/Agent/Robotics 중심 PoC 수행<br>- 현장 성과지표 검증 및 개선  | - 주요 현장 PoC 수행 (예: Safety AI, 일정관리 Agent)<br>- ROI 및 효율지표 정량화 | Build (모델 협력), Alpha (예산/PoC 관리)           |
| **3단계 (Q3-Q4)**<br>“내재화 및 확산”     | - CQSS+I 통합관리체계 구축<br>- 사례 매뉴얼화 및 타현장 확산                | - 통합 Dashboard PoC<br>- 성공사례 정리 및 전사 확산세션 개최                  | Foundation (데이터 관리), Alpha (문화 확산)         |

## 협업 구조
| 축                                | 주요 파트너                            | 협력 형태                 | 목적                        |
| -------------------------------- | --------------------------------- | --------------------- | ------------------------- |
| **AI Board 내 파트 협력 (내부 연계)**         | Alpha, Build, Foundation          | 기획·데이터·기술 연계          | Edge는 PoC 현장 실증 및 피드백 담당  |
| **타 부서/조직 협력 (Cross Collaboration)** | DT/DX 팀, 품질/안전/생산성 센터, 각 본부 기술팀   | 공동과제 기획, 현장 운영 리소스 확보 | Edge 인력 부족 보완 + 현장 실행력 확보 |
| **현장 조직 협력 (현업 연계)**                 | 주요 시공현장, 현장 기술팀(공무, 품질, 안전), 협력업체 | 현장 실증, 데이터 수집, PoC 운영 | 실제 검증 데이터 확보 + 현장 내재화 촉진  |

## 협력 프로토콜 (운영 방식)

| 구분                               | 설명                            | 실제 예시                                       |
| -------------------------------- | ----------------------------- | ------------------------------------------- |
| **공동 과제체계 (Joint Task)**         | 각 부서와 공동으로 과제 정의 및 추진         | “Edge-품질본부 공동 PoC: AI 기반 균열검사 자동화”          |
| **AI 현장 TF 구성 (Site Taskforce)** | 주요 현장별 담당자 지정 + Edge 리드       | “○○현장 AI TF: Edge 1명 + 품질담당 1명 + 기술지원 1명”   |
| **PoC 파트너십 운영 (Tech Partner)**   | 외부 기술사와 공동검증, Build 협력        | “DeepRobotics, Namuh 등과 Vision/Robotics 실증” |
| **데이터 루프 거버넌스 (Foundation 연계)**  | 현장 데이터 → Data Lake → 분석 → 피드백 | “Edge→Foundation 간 E2E Dataflow 운영체계 구축”    |
