# 개요

## n8n이란?

- **n8n (Node for Node)**: 오픈소스 자동화 워크플로우 툴
- **형태**: 노코드 기반 + 시각적 인터페이스
- **통합성**: 400개 이상의 외부 앱(API)과 연동 가능 (Google, Slack, Notion 등)
- **호스팅 방식**:
  - 클라우드(n8n.io)
  - 셀프 호스팅(Docker, 서버 설치 등)

# 내용


## n8n의 AI 기능 개요

### 1. **AI 노드 기반 워크플로우 (AI Chain)**
- 고정된 절차 내 AI 노드를 포함
- 예시: PDF 파일 요약 → Slack 메시지 발송
- 특징: 안정적, 예측 가능, 반복 업무에 적합

### 2. **AI 에이전트 기반 워크플로우**
- AI가 순서 및 툴 사용 판단
- 대화형 요청에 따라 동적 작업 수행
- 사용 도구:
  - **LLM (ChatGPT, Claude, Gemini 등)**  
  - **Memory**: 최근 대화 기억
  - **Toolset**: Gmail, Google Sheets, Calendar 등과 연결

---

## LLM 통합 구조

| 구성 요소 | 설명 |
|------------|------|
| **LLM (두뇌)** | GPT-4, Claude, Gemini 등 연결, API Key 필요 |
| **Memory (기억)** | 최근 대화 컨텍스트 유지 |
| **Tools (손발)** | 캘린더, 메일, 시트 등 외부 앱 활용 |
| **System Prompt** | 도구 활용 방식 정의, 오늘 날짜 삽입 가능 |

> 예: `${now:YYYY-MM-DD}` → 날짜 자동 반영

---

## 활용 시나리오

| 시나리오 | 설명 |
|----------|------|
| 일정관리 AI 비서 | 일정 조회, 생성, 삭제, 참석자 초대, 리마인더 전송 |
| 문서 요약 자동화 | Google Drive 문서 요약 후 Slack 발송 |
| VOC 분석 | 이메일 수신 → GPT 분석 → DB 저장 |
| Slack 챗봇 | 질문 → AI 응답 → 자동화 연결 수행 |

## 호스팅 방식 비교

| 항목 | Self-hosting | n8n Cloud |
|------|--------------|-----------|
| 비용 | 무료 (서버 필요) | 유료 (서브스크립션) |
| 보안 | 직접 통제 가능 | 외부 서버 사용 |
| 커스터마이징 | 높음 | 낮음 |
| 적합 대상 | 기업, 엔지니어링 조직 | 일반 사용자, 빠른 시도 목적 |

## MCP와의 비교

| 항목           | **n8n**                   | **MCP (Microsoft Copilot Platform)** |
| ------------ | ------------------------- | ------------------------------------ |
| **기반 구조**    | 노드 기반 오픈소스 자동화 툴          | 마이크로소프트 365 기반 Copilot 통합 플랫폼        |
| **AI 연결**    | GPT, Claude, Gemini 등 선택형 | 주로 OpenAI GPT 기반, M365 전용 최적화        |
| **사용 범위**    | 범용 오토메이션, 커스텀 가능          | 워드, 엑셀, 팀즈 등 오피스 생태계 전용              |
| **자동화 스타일**  | 툴과 조건을 조합해 복잡한 자동화 구성 가능  | 주로 자연어 기반의 워크플로우 단순화                 |
| **확장성**      | 고도화된 워크플로우 구현에 유리         | M365 안에서 제한적 자동화 제공                  |
| **비용**       | 무료 셀프 호스팅 가능              | 유료 (Microsoft 365 Copilot 라이선스 필요)   |
| **기술 진입 장벽** | 중간 (API 연동, 인증 설정 등 필요)   | 낮음 (MS 계정 기반 활용)                     |

> 📌 **요약**:  
> - **n8n**은 유연성과 커스터마이징이 뛰어나며, 다양한 SaaS와 통합 가능한 자동화 플랫폼  
> - **MCP**는 MS 오피스 사용자를 위한 AI 보조 도구로, 조직 내 문서 중심 업무에 최적화됨  
> - 두 시스템은 **용도, 대상 사용자, 접근성** 측면에서 서로 보완적임

##  기술 세부 사항

### 인증 방식
- Google API 연결 시:
  - OAuth 동의화면 설정
  - Client ID/Secret 발급
  - Scope 설정 (calendar, sheets, gmail, drive 등)
  
### Webchat 인터페이스
- n8n Webhook + ChatUI로 "나만의 챗봇" 구축 가능
- 인증 로그인, 환영 메시지, 채팅 기록 자동화 가능

---

## 장단점 요약

### 장점
- 비개발자도 노코드로 자동화 및 AI 도입 가능
- LLM + 다양한 툴 조합 가능 (확장성 뛰어남)
- 시각적 인터페이스 및 로그 추적 기능 우수
- 셀프 호스팅 가능 (비용/보안 측면 유리)

### 단점
- OAuth 인증 등 초기 설정 복잡
- 툴이 많아지면 처리 속도/정확도 저하 우려
- 프롬프트 설계 역량에 따라 성능 편차 발생
