# 개요

Phase 2는 Mini PC를 “일반 Windows PC”에서 “홈랩 서버(하이퍼바이저)”로 전환하는 단계이다.  
이 단계에서 기존 Windows 11은 제거되고, Proxmox VE가 디스크에 새로 설치된다.

본 단계의 목표는 다음과 같다.

- 디스크 초기화(Windows 제거) 및 Proxmox VE 설치 완료
    
- Proxmox 웹 UI 접속 가능 상태 확보
    
- 네트워크(고정 IP 또는 DHCP 예약) 기반의 안정적 운영 기반 마련
# 내용

## 2.1 시작 전 최종 체크

### 2.1.1 되돌릴 수 없는 변경사항 확인

- Proxmox 설치 과정에서 설치 대상 디스크는 초기화된다.
    
- Windows 11 및 내부 데이터는 모두 삭제된다.
    

### 2.1.2 필요한 준비물

- Proxmox 설치 USB
    
- 유선 LAN 연결(권장)
    
- 공유기 관리자 접근(고정 IP 또는 DHCP 예약 목적)
    
- 설치 중 사용할 키보드/모니터(최초 1회 필요)
    

---

## 2.2 BIOS 부팅 및 USB로 부팅

### 2.2.1 Boot Menu 진입

- 전원 켜기 직후 Boot Menu 키(F12/F10/ESC/DEL/F2 등) 연타
    
- USB 부팅 항목 선택
    

### 2.2.2 부팅 모드 확인

- UEFI 모드로 USB 부팅되는지 확인
    
- 만약 “Legacy/CSM”으로만 보이면 BIOS에서 CSM 관련 옵션 확인 필요
    

---

## 2.3 Proxmox VE 설치 진행

### 2.3.1 설치 시작

- Proxmox 설치 화면에서 “Install Proxmox VE” 선택
    

### 2.3.2 EULA 동의

- 동의 후 다음 단계 진행
    

### 2.3.3 설치 디스크 선택 (Windows 포맷이 여기에서 수행됨)

- 설치 대상 SSD를 선택하는 순간 “Windows가 들어있던 디스크를 Proxmox가 사용할 디스크로 지정”하는 절차가 된다.
    
- 설치 디스크 선택 전에 디스크 용량/모델명을 확인하여 다른 디스크를 선택하지 않도록 주의한다.
    

### 2.3.4 파일시스템 선택 기준 (권장 포함)

#### 옵션 A: ext4 (권장: 입문/단일 디스크/단순 운영)

- 장점
    
    - 단순하고 문제 발생 시 대응이 쉬움
        
    - 단일 SSD 환경에서 운영 안정성 높음
        
- 단점
    
    - ZFS의 스냅샷/무결성 기능은 제한적(운영 방식에 따라 충분히 커버 가능)
        

#### 옵션 B: ZFS (single disk) (권장: 학습/스냅샷 활용/메모리 여유 있을 때)

- 장점
    
    - 스냅샷/무결성/관리 편의성 강점
        
- 단점
    
    - 메모리 사용량이 증가할 수 있음(16GB면 가능하나, VM 운용 규모를 관리해야 함)
        
    - 단일 디스크에서는 ZFS의 미러/RAID 강점을 활용하기 어려움
        

#### 추천 결론 (N100 + RAM 16GB + 단일 SSD 기준)

- 운영 단순성과 안정성을 우선하면 ext4
    
- 스냅샷 중심 운영/학습이 목적이면 ZFS(single disk)

## 2.4 지역/시간/키보드 설정

- Time zone은 Asia/Seoul로 맞춘다.
    
- 키보드는 사용 환경에 맞게 선택한다(예: Korean, US 등).

## 2.5 관리자 계정 및 이메일 설정

### 2.5.1 Password 설정

- Proxmox root 비밀번호 설정
    
- 추후 웹 UI 로그인에 사용됨
    

### 2.5.2 Email 입력

- 경고/알림 수신 목적
    
- 실제 사용 가능한 이메일 권장(필수는 아님)
    

---

## 2.6 네트워크 설정 (중요)

### 2.6.1 호스트명(Hostname) 설정

- 예시: `pve-home` 또는 `hyungjuns-pve`
    
- 도메인 형태로 요구될 수 있음(예: `pve-home.local`)
    

### 2.6.2 IP 설정 방식 선택

#### 방식 A: 고정 IP를 설치 단계에서 직접 지정(권장)

- 장점
    
    - 서버 운영에 가장 안정적
        
    - 원격 접속 주소가 변하지 않음
        
- 설정 항목
    
    - IP Address
        
    - Netmask
        
    - Gateway
        
    - DNS
        

#### 방식 B: DHCP 사용 후 공유기에서 DHCP 예약(대안)

- 장점
    
    - 설치가 빠름
        
- 단점
    
    - 초기에는 IP가 바뀔 수 있음
        
    - 공유기 설정을 반드시 진행해야 안정화됨
        

#### 추천 결론

- 가능하면 설치 단계에서 고정 IP를 지정한다.
    
- 공유기 설정 권한이 없다면 DHCP로 설치 후, 가능한 시점에 DHCP 예약으로 전환한다.
    

---

## 2.7 설치 완료 및 재부팅

### 2.7.1 설치 완료 후 USB 제거

- 설치 완료 안내가 나오면 재부팅 전에 USB를 제거하거나,
    
- 재부팅 직후 부팅 순서에서 USB가 다시 잡히지 않도록 한다.
    

### 2.7.2 Proxmox 부팅 확인

- Proxmox 로그인 프롬프트가 뜨면 정상
    
- 화면에 IP 및 웹 UI 주소가 표시될 수 있음
    

---

## 2.8 Proxmox 웹 UI 접속 및 초기 확인

### 2.8.1 웹 UI 접속

- 같은 네트워크의 PC에서 브라우저로 접속
    
    - `https://<Proxmox_IP>:8006`
        

### 2.8.2 인증서 경고 처리

- 자체 서명 인증서로 인해 브라우저 경고가 뜨는 것이 정상
    
- 고급(Advanced)에서 접속 진행
    

### 2.8.3 로그인

- 사용자: `root`
    
- 비밀번호: 설치 시 설정한 root 비밀번호
    

---

## 2.9 초기 설정 (설치 직후 필수 권장 작업)

### 2.9.1 업데이트

- Proxmox 패키지 업데이트 적용
    
- 커널/패키지 업데이트 후 필요 시 재부팅
    

### 2.9.2 저장소(repo) 설정

- 커뮤니티(Non-subscription) 저장소 사용 설정(개인 홈랩 일반적)
    
- 엔터프라이즈 저장소 관련 경고가 있으면 정리
    

### 2.9.3 시간 동기화 확인

- 시간대 및 NTP 동기화 확인(Asia/Seoul 기준)
    

### 2.9.4 기본 네트워크 브릿지 확인

- `vmbr0` 존재 및 물리 NIC에 연결되어 있는지 확인
    
- 이후 VM이 외부 네트워크로 나갈 때 핵심 구성
    

---

## 2.10 Phase 2 완료 기준 (성공 체크)

- Proxmox VE가 SSD에 설치되어 정상 부팅된다.
    
- 웹 UI에 접속 가능하다.
    
- 서버 IP가 고정되어 있거나(DHCP 예약 포함) 예측 가능하다.
    
- 업데이트 및 기본 설정이 완료되었다.

# 요약

Phase 2는 Windows 11 환경을 제거하고 Proxmox VE를 설치하여 Mini PC를 서버로 전환하는 단계이다.  
핵심 포인트는 아래 3가지다.

- 설치 디스크 선택 = Windows 포맷 및 서버 전환이므로 디스크 선택을 정확히 한다.
    
- 파일시스템은 단일 SSD 기준으로 ext4 또는 ZFS(single disk) 중 목적에 맞게 선택한다.
    
- IP는 가능한 고정으로 확보하여 원격 접속과 운영 안정성을 보장한다.

위 단계가 종료되었다면 다음 단계인 [[Phase 3. Proxmox 초기 정비 및 Ubuntu Server VM 생성]] 를 진행한다.

- 스토리지 구조 확인(ISO 저장소, VM 디스크 위치)
    
- 네트워크 브릿지(vmbr0) 점검
    
- Ubuntu Server ISO 업로드
    
- Ubuntu VM 생성(권장 리소스 설정 포함)
