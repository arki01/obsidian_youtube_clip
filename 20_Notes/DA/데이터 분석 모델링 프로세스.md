# 개요
데이터 분석 모델링을 위해서 아래와 같은 프로세스로 진행되며, 실제로 데이터분석기사에서도 아래 순서로 분석 테스트를 진행한다

# 내용

#### 모델링을 위한 데이터 마트 설계와 구축
- 모델링 작업 이전에 데이터 마트를 설계해야한다. 기존 정보 시스템 내의 데이터를 최대한 활용하고, 채택된 가설을 기반으로 마트를 설계해야한다. 
#### 탐색적 분석(EDA)과 유의 변수 도출
- 데이터 마트가 구축되면 구체적인 사실을 발견해 통찰을 얻기 위해 탐색적 분석(EDA : exploratory data analysis)을 수행한다. EDA는 시간이 매우 많이 들어가며, 최근에는 EDA를 자동으로 신속하게 수행하여 유의미한 값만 파악해 데이터 마트로 만든 후 모델링 업무를 수행하는게 일반적이다.
#### 모델링(모형화)
- 모델링을 위한 위 두 단계 준비가 끝났다면, 문제의 단순화를 통해 변수 간의 관계로 정의해야 한다. 이를 모델링(모형화)라고 한다. 예측하고자 한다면 회귀분석, 회귀나무분석 등의 모델을 사용할 수 있고, 분류를 하고자 한다면 knn, 로지스틱 회기분석 등을 사용할 수 있다. 
#### 모델링 성능 평가
- 각 모델에 따라 그 방법이 다르며 이 부분은 추가 보완하겠다.
#### 검증 및 테스트
- 분석용 데이터를 학습용과 검증용, 테스트용으로 분리한다음 분석용 데이터를 이용해 자체 검증한다. 데이터 분할 비율은 전체 데이터가 충분할 경우 학습용:검증용:테스트용 비율을 6:2:2로 일반적으로 선정할 수 있다. 
#### 적용
- 분석결과를 업무 프로세스에 완전히 통합해 실제 일,주,월 단위로 운영하는 단계다. 분석시스템과 연계해 사용할 수 있고, 별도 코드로 분리해 기존 시스템에 별도 개발해 운영할 수 있다.
