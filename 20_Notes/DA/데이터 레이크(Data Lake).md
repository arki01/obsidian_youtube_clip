# 개요
방대한 데이터를 ETL과 DW구축의 한계에 다다랐고, 전통적인 데이터 뿐만 아니라 수많은 비정형 데이터를 실시간으로 수집, 정제, 통합, 분석해 활용하기 위해 '데이터레이크' 라는 새로운 개념의 플랫폼을 도입하고 있다.

이에 대한 데이터레이크의 특징과 데이터웨어하우스의 차이점을 알아보고자 한다.

# 내용
#### 특징
- 데이터 레이크는 빅데이터에 최적화되어 있다. 
- 데이터 레이크에서는 데이터(Raw Data) 형태 그대로 적재된다.
- ETL 과정에서 처럼 데이터 스키마를 맞추는 과정은 필요하지 않다.
- 다만, Raw 데이터 임에 따라 데이터 레이크에 직접 엑세스 하여 분석하고 관리하기에는 적합하지 않다.
#### 데이터 웨어하우스와 데이터 레이크 비교
- 데이터 웨어하우스 : 리포팅(보고)를 위해 설계된 구조환 데이터 모델을 제공한다. 기본적으로 정제된 데이터로 자체의 스키마가 존재한다. 이에 따른 ETL과정이 필요하고, 제한적인 범위에서만 활용이 가능하다. 그리고 데이터 조작이나 가공을 위해 IT부서에 의존할 수 밖에 없다.
- 데이터 레이크 : 정형, 비정형을 막론한 다양한 형태의 Raw Data를 모은 집합 개념으로, 데이터 분석가, 데이터 사이언티스트, 개발자 등의 사용자들이 인사이트를 얻기 유용하다.
