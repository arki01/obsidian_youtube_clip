# 개요
빅데이터 분석을 수행하기 이전 과제를 발굴하기 위한 방법론을 이야기한다.
크게 하향식과 상향식으로 나누어지며, 분석대상의 인지 유무에 따라서 결정할 수 있다.
# 내용
#### 하향식 접근법
- 하향식 접근법 : 분석대상을 알고 있을 경우 
	- 문제 탐색 단계 : 여러가지 방법을 통해 현재 어떠한 비지니스에서 어떠한 문제가 있는지 파악하는 단계
	- 문제 정의 단계 : 비지니스 문제가 식별되면 이를 데이터의 문제로 변환한다. 예를 들어 '고객의 불만에 영향을 끼치는 요인은 무엇인지 분석하고, 그 요인과 고객 불만율에 대한 상관 및 예측 모델을 수립한다.' 등이 있겠다.
	- 해결방안 탐색 단계 : 먼저 기존 시스템으로 가능한지, 기업의 역량이 있는지 판단하여 해결방안을 선정한다.
	- 타당성 검토 : 경제적 타당성과 데이터 및 기술적 타당성을 검토한다.

#### 상향식 접근법
- 상향식 접근법 : 분석대상을 모르고 있을 경우
	- 지도,비지도학습 
		- 지도학습 : 정답이 있는 데이터를 활용하여 분석모델을 학습시키는 것. 레이블(Label)이 범주형인 분류와 연속형인 회귀로 나누어 진다.
		- 비지도학습 : 지도학습과 달리 정답을 알려주지 않고  학습하는 것이다. 정답이 없는 데이터를 비슷한 특징을 가진 데이터와 군집화 하여 새로운 데이터에 대한 결과를 예측한다. 
	- 프로토타입 접근법
		- 문제 정의가 불명확할 경우 먼저 분석을 시도하고 그 결과를 확인하면서 조금씩 개선하는 법이다.
