# 개요
데이터 웨어하우스는 사용자의 의사결정에 도움을 주기 위해 기간시스템의 데이터 베이스에 축적된 데이터를 공통의 형식으로 변환해서 관리하는 데이터베이스를 말한다. 특징으로는 주제별로 통합하여 운영되며 혼재한 데이터를 통합하는 특징을 갖는다.

# 내용
### 개요
- 데이터 웨어하우스는 사용자의 의사결정에 도움을 주기 위해 기간시스템의 데이터 베이스에 축적된 데이터를 공통의 형식으로 변환해서 관리하는 데이터베이스를 말한다. 
- BI (Business Intelligence) : DW는 BI를 필수적으로 동반한다. 경영진 또는 사용자의 의사결정을 위해서 데이터를 통합관리하는 데이터웨어하우스가 필요하며, 이를 분석 및 가공하여 개발자가 아닌 경영진이 보기 쉽게 시각화하여 보여주는 프로세스가 BI다. 
### 특징
- DBMS의 한계 : DBMS는 신속하고 정확하게 시장을 분석해서 의사결정의 기초로 활용하기에는 한계가 있었다. 이는 OLTP(OnLine Transaction Processing) 시스템이 가지는 한계이기도 했다. 그래서 DW를 구축하고 OLAP(OnLine Analytical Processing) 시스템 기반의 BI 프로세스를 수행하는 쪽으로 데이터 관리는 변화하게 된다.
- DW의 특징 : 많은 데이터들을 주제별로 톡합하여 운영시스템과 사용자 사이의 별도 저장해 놓은 데이터 베이스 개념이다. 원본데이터는 훼손하지 않으며 오로지 의사결정을 위한 리포팅에 포커싱을 둔다.
### 구성
- DW는 ERP, CRM, 공급망관리(SCM) 등 기업에서 활용되는 다양한 시스템의 데이터를 한곳에 담아두는 방식이다.
- DW는 이렇게 모아둔 데이터를 Reporting, Budgeting, Dashboards 로 제공하여 의사결정을 지원하는 중간자 역할을 한다.
### 테이블 모델링 기법
#### 스타스키마
- 조인 스키마라고도 하며 단일 테이블을 중심으로 다수의 차원 테이블이 연결돼 있다.
- 복잡도가 적으며 이해하기 쉬우나, 차원 데이블들의 비정규화로 상대적으로 데이터 적재에 시간이 많이 소요된다.
#### 스노우플레이크
- 스타스키마의 차원 테이블이 제3 정규형으로 정규화된 형태이다.
- 테이블은 그대로 유지되는데, 꼬리에 꼬리를 물고 테이블이 계속 등장한다.
- 저장공간은 최소화되고 유연성은 증가하지만, 복잡하고 결과 검증이 어렵다.

### 처리 프로세스
#### ETL
- ETL : Extraction, Transformation and Load
- 데이터 원천으로부터 추출 및 변환하여 데이터웨어하우스, 데이터마트에 적제하는 작업을 말한다. 

#### ODS
- ODS : Operational Data Store
- 다양한 원천 데이터로 부터 데이터를 추출 통합한 데이터베이스이며, 향후 다양한 보고서 생성을 위해 데이터 웨어하우스로 이관된다.
#### CDC
- CDC : Change Data Capture
- 데이터 변경을 식별해 필요한 후속 처리(데이터 전송/공유 등)를 자동화하는 기술을 말한다. 
#### EAI
- EAI : Enterprise Application Integration
- 기업 정보 시스템들의 데이터를 연계 통합하는 소프트웨어 및 정보 시스템 아키텍처 프레임워크
- 산재한 정보 시스템들을 프로세스 및 메세지 차원으로 통합 관리할 수 있게 된다.
